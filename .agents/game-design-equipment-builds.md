# Game Design: Equipment Build System (Core / Support Affixes)

Tracking issue: y-krn/wiz-mobile-rpg#120

# 概要

ビルド多様性の欠如（実効ビルド数 ≈ 職業選択8通りのみ）を解消するため、装備を「コア」「サポート」の2階層アフィックスに再設計する。シナジーシステム（`SYNERGIES` / `getActiveSynergyMod`）は廃止済み前提（別Issue化しない場合は Phase 1 に含める）。

目標: 実効ビルド空間 ≈ 16コア × 職業適性2〜3 × サポート構成 ≈ 60〜80（現状の約10倍）。

# 全体構造

- **コア16種**: ルールを変える効果。ドロップ限定（門番・中ボス・ボス確定枠、B3F以深宝箱、レア敵）。商店・行商・工房からは入手不可。1キャラにつきコア装備は1個まで。
- **サポート40種**: 数値・小効果。刻印（工房）で付与・上書き可能。
- 純粋な数値上位のコアは作らない。全コアはサイドグレード。

- 基礎数値 17（現行維持）: str/int/pie/vit/agi/luk, hp/mp, antiUndead/antiDragon/antiDemon, poisonWard, spellGuard, trapBonus, treasureSense, arcaneSense, hearRange, traceRead
- 条件数値 12（新設）: 深層(B3F以深)攻+ / 前列防+ / 後列回避+ / HP満タン時与ダメ+ / 1ターン目攻+ / 対beast+ / 対spirit+ / 先制時防+ / 単独生存時全ステ+ / 疲労中ペナルティ半減(疲労実装まで保留可) / 状態異常耐性+ / 呪文命中+
- トリガー小 6（新設）: キル時HP2回復 / 追撃時MP1回復 / 罠解除時小額G / 被弾時稀に怯み / 勝利時稀に素材 / 階段発見時HP回復

# 生成・入手規則

- コモン=サポート1 / レア=サポート2 or コア1 / エピック=コア1＋サポート2
- コアドロップの30%は呪い付き。封印すると呪い除去＋コア効果半減
- フロア別コアプール: 浅層=経済系中心、深層=戦闘系

# バランス枠組み

- ポイント予算制: サポート1〜3点、コア一律10点。アイテム予算=レアリティ×フロア深度
- コアは期待稼働率換算で無条件+15%相当を上限
- 調整つまみ: コアドロップ率 / 呪い付与率30% / 換算上限15%

# UI

- 装備画面: コアはアイコン＋条件文1行。戦闘ログでコア発動を必ず明示（例:「[背水] 刃が燃え上がる」）
- 図鑑にコア発見録（16種コレクション）

# フェーズ分割

- Phase 1: 基盤（アフィックス2階層スキーマ、予算制生成、シナジー廃止、セーブ移行、設計ドキュメント）
- Phase 2: 戦闘系コア10＋戦闘系サポート
- Phase 3: 経済・探索系コア6＋経済系サポート、半鑑定表示
- Phase 4: 工房拡張（刻印20種、研磨、封印のコア半減）

各フェーズは個別Issue・個別PRで実施。
