# 設計: 宝箱罠の非盗賊救済（叩き壊す＋罠外しキット）

日付: 2026-07-19
状態: 承認済み（設計のみ。実装は別途指示）

## 背景と問題

宝箱の罠は事実上盗賊専用機能になっている。

- 調査成功率: 盗賊85% / 他30%（失敗時は偽の罠名を表示し、偽と気づけない）
- 解除成功率: 盗賊85% / 野伏60% / **他全員25%（忍者含む）**
- 罠補正装備 `THIEF_EYE`(+10%)は盗・野・忍専用で非盗賊救済にならない
- B4/B5は罠100%。テレポーターは進行崩壊級
- 非盗賊（特にソロ戦士/魔/僧ラン）の実質選択肢は
  「罠を食って開ける」か「立ち去る」の二択で、意思決定が消滅している

対照的に床罠（`src/systems/traps.js`）は
LUK+AGI ベース成功率＋「引き返す/迂回/強行/解除」の4択で全職に選択肢がある。
宝箱だけがこの設計思想から取り残されている。

**主な救済対象はソロ非盗賊ラン**（ソロ突破率平準化の流れ、#173 系）。

## 是非論の結論（記録用）

罠・宝箱は存続。リスクリワード判断と盗賊の存在意義の核。
欠陥は罠そのものではなく「非盗賊にまともな悪い選択肢すら無い」こと。
本設計は全職に**悪いが実行可能な選択肢**を与え、4段構造にする:

| 手段 | コスト | 対象 |
| --- | --- | --- |
| 盗賊/野伏/忍者の解除 | 無償（失敗リスク小） | 該当職 |
| 罠外しキット | 素材コスト・確定 | 全職 |
| 叩き壊す | 弱体罠被弾＋消耗品破損リスク | 全職 |
| 立ち去る | 機会損失 | 全職 |

## 機能1: 叩き壊す（新ボタン）

宝箱メニューに「叩き壊す」を追加。全職・調査不要・いつでも選択可。

動作:

1. 罠がある場合、**弱体化して発動**（床罠 `weakenedModifiers` の思想を流用）:
   - 毒針: ダメージ6（通常12の半減）、毒付与50%
   - ガス爆弾: 全体2〜6（通常5〜12の半減）
   - 閃光弾: 盲目率30%（通常60%の半減）
   - テレポーター: **50%で不発**。発動時は通常転移
     （転移効果は半減不能のため発動率側を半減）
2. 報酬のうち **usable タイプの `chest.item` のみ30%で破損**
   （ログ:「衝撃で中身の一部が砕けた…」）。
   装備・装身具（`accessoryItem`）・素材束・鑑定粉は無事。
3. 破損判定と罠発動は独立に解決。宝箱は必ず開く。

盗賊の価値維持: 解除成功なら無傷・無破損。叩き壊すは常に「痛みを買う」選択。

## 機能2: 罠外しキット（新消耗品 `TRAP_KIT`）

- `src/data/items.js` に追加。type: "usable"、**職制限なし**
- 宝箱メニューに「キットで解除」ボタン（**所持時のみ表示**）。
  使用で1個消費し罠を確定解除（`trap = "none"`）。調査不要
- 解除カウント系の扱い: `trapsDisarmed` は加算**しない**
  （盗賊スキルによる解除と区別。CORE_TRAP_EATER の対象外）
- 入手経路:
  - マイルストーン商人（`src/data/milestone_merchant.js`）: `骨片2`（暫定価格）
  - B2〜B5 宝箱候補プール（`src/chest.js` candidates）に追加
- B1はキット入手不可だが、罠35%無し＆軽症罠のみで問題なし
- 価格・出現率はバランスsimで調整前提

## 機能3: 忍者の宝箱解除率修正（同梱推奨・分離可）

`executeDisarm` の忍者を 0.25 → **0.70**。
床罠は盗>忍>野の序列（Thief lvl*2+15 / Ninja lvl*1.5+10 / Ranger lvl+5）なのに
宝箱では忍者=戦士扱いの不整合を解消。序列: 盗0.85 > 忍0.70 > 野0.60。

## UI配置（モバイル・1カラム維持）

上から:
